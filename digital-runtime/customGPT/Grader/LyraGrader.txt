Thought for 16s

Mogri usually refers to green radish seed pods, also called moongre/moongre ki sabzi in North Indian cooking. They are the edible pods of the radish plant, not the root. They look like thin green beans, taste mildly peppery, and are often cooked as a dry sabzi with potatoes and spices.

In food use, mogri is treated as a winter vegetable, commonly available around December to February in parts of India. It can be sautéed, pickled, made into chutney, or used raw when tender.

The word can mean other things too. In Urdu/Hindi dictionary contexts, mogri may mean a rammer/pestle-like tool, while in some dictionary entries it is also linked to Arabian jasmine, so the exact meaning depends on region and context.

Most likely, though, when someone says “mogri” in a market or recipe, they mean radish pods.

Yeah — DRAGI is the partner piece to MOGRI.

From the public posts/GitHub I found, DRAGI = DragonRuntime, and the creator frames it like this: “Mogri holds the object, Dragi defines the object.” It is niche, not a mainstream prompt-engineering standard. The GitHub README describes the core as four object-defining questions: what it eats, where it lives, how you call it, and what eats it.

Clean promptbot read:

MOGRI = preserve the object across prompts.
DRAGI = define the object so the model knows what it is, where it operates, what feeds it, what threatens it, and what boundaries apply.

A usable DRAGI shape:

DRAGI object definition:

Object:
[What is the thing/process/prompt/system being defined?]

Eat:
[What inputs, fuel, evidence, context, or data does it consume?]

Loc:
[Where does it live or operate? Chat, codebase, agent runtime, document, workflow, user task?]

ID:
[What is its name, handle, interface, or stable identifier?]

Eater:
[What consumes, evaluates, constrains, attacks, or depends on it?]

Foes:
- Beast: the object becoming uncontrolled or misread
- Best: over-optimization that changes the object
- Post: output drift after the object is defined
- Pest: noise, interference, distraction, bad inputs

Controls:
- Law: rules it must obey
- Wall: boundaries it must not cross
- Roar: visible output behavior
- War: conflict/adversarial pressure handling

Fixed:
Do not redefine the object unless explicitly instructed.

The short version in human language:

DRAGI asks: “What creature is this?”
What feeds it, where it lives, what name it answers to, what can kill/eat/corrupt it, and what rules cage it.

So for AI prompting:

Use MOGRI when the model keeps losing the core object.
Use DRAGI when the object itself is fuzzy and needs definition.
Use both when building long prompts, agents, grading systems, custom GPTs, or workflows where the model starts swapping targets.
